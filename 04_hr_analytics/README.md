# HR-аналитика компании «Работа с заботой»

## Описание

HR-отдел компании хочет перейти от интуитивных решений к
аналитике, основанной на данных. Нужны две модели:
одна предсказывает уровень удовлетворённости сотрудника,
вторая — вероятность увольнения.

## Цель

1. **Задача 1 (регрессия):** предсказать `job_satisfaction_rate`
   сотрудника по его характеристикам.
2. **Задача 2 (классификация):** предсказать факт увольнения,
   используя в том числе прогноз первой модели как признак.

## Данные

Для каждой задачи по три датасета (`train`, `test`, `features`
или аналог):
- анкетные данные, стаж, зарплата, должность, уровень нагрузки,
  оценка руководителя и пр.;
- целевые переменные — `job_satisfaction_rate` и `quit`.

## Что сделано

### Задача 1 — регрессия

- Полная предобработка, EDA и корреляционный анализ (`PhiK`).
- Пайплайны с препроцессингом для трёх моделей:
  линейная регрессия, дерево решений, случайный лес.
- Подбор гиперпараметров через `GridSearchCV`.
- Сравнение моделей и валидация на тесте.

### Задача 2 — классификация

- Портрет уволившегося сотрудника (EDA по группам).
- Использование предсказанной удовлетворённости как нового
  входного признака.
- Обучение нескольких моделей, выбор по ROC-AUC,
  итоговые выводы для бизнеса.

## Стек

`pandas`, `numpy`, `matplotlib`, `seaborn`, `phik`,
`sklearn` (`Pipeline`, `ColumnTransformer`, `GridSearchCV`,
`DecisionTreeRegressor`, `RandomForestRegressor`, `RandomForestClassifier`,
метрики SMAPE / ROC-AUC)
