# Предсказание оттока клиентов оператора связи «ТелеДом»

## Описание

Финальный проект курса. Оператор связи «ТелеДом» хочет заранее
находить клиентов, планирующих расторгнуть договор, чтобы
предлагать им промокоды и удерживать. Задача — обучить модель
бинарной классификации, которая по профилю абонента и его
услугам предсказывает отток.

## Цель

Построить модель с **ROC-AUC ≥ 0.85** на тесте.

## Данные

Четыре таблицы:
- `contract_new.csv` — информация о договоре (даты начала и
  окончания, тип оплаты, ежемесячные / итоговые расходы);
- `personal_new.csv` — персональные данные клиента;
- `internet_new.csv` — интернет-услуги;
- `phone_new.csv` — услуги телефонии.

## Что сделано

- EDA и предобработка каждой таблицы по отдельности, затем
  объединение в единый датафрейм.
- Конструирование целевой переменной из поля `EndDate` (дата
  расторжения договора).
- Анализ баланса классов, распределений признаков,
  корреляционный анализ через `PhiK`.
- Обучение трёх моделей через единый пайплайн:
  - логистическая регрессия,
  - случайный лес с подбором гиперпараметров,
  - **градиентный бустинг** с подбором гиперпараметров.
- Сравнение по ROC-AUC на кросс-валидации, выбор лучшей модели,
  финальный замер на hold-out.
- Итоговые рекомендации заказчику.

## Стек

`pandas`, `matplotlib`, `phik`, `sklearn` (`Pipeline`,
`ColumnTransformer`, `GridSearchCV`, `LogisticRegression`,
`RandomForestClassifier`, `GradientBoostingClassifier`,
`roc_auc_score`)
